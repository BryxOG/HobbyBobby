# HobbyBobby — монорепо .

Поиск людей по интересам и организация ивентов. Фронт и микросервисы — папками
верхнего уровня, каждая со своим стеком и сборкой.

```
frontend/     Next.js PWA. Весь UI, работает на моках. См. frontend/AGENTS.md
docs/         Общая документация: спека MVP и эскиз UI (источник истины)
```

Позже рядом появятся `event-service/`, `user-service/`,
`notification-service/` (Java Spring + Python).

## Правила монорепо

- **Работаешь с фронтом — сначала прочитай `frontend/AGENTS.md`.** Там стек,
  архитектура и грабли, на которые уже наступали.
- **Команды запускать из папки сервиса**, не из корня. Для фронта:
  `cd frontend && npm run dev`. В корне нет ни `package.json`, ни общего
  рантайма — это не npm workspaces, а просто соседние папки.
- **`.gitignore` — свой в каждой папке.** В корневом только ОС-мусор, секреты и
  артефакты Java/Python. Паттерны с ведущим слешем (`/node_modules`) якорятся к
  папке своего `.gitignore`, поэтому его нельзя поднимать в корень.
- **Контракт между фронтом и бэком —
  `frontend/src/lib/api/types.ts`.** Это готовые DTO; новые сервисы равнять по
  ним, а не изобретать свои формы ответов. Точка подключения —
  `frontend/src/lib/api/client.ts`, там одна строка.
- **Деплой фронта на Vercel: Root Directory = `frontend`**, иначе сборка не
  найдёт `package.json`.

## Источник истины по UI

`docs/sketch.excalidraw` (34 экрана, светлая и тёмная темы) плюс спека
`docs/EVENTS_SERVICE_MVP.md`. Где расходятся — эскиз главнее: в спеке нижняя
навигация описана как «Главная/Карта, Чаты, Создать, Поиск/компас, Профиль», но
стрелки эскиза ведут в **Ивенты, Чаты, Создать(+), Карта, Профиль**. Сделано по
стрелкам; в спеке этот пункт стоит поправить.
