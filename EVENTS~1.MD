# EventService — требования к MVP

Документ собран на основе исходных заметок и уточняющих ответов. Служит рабочей базой для системного дизайна (делается параллельно) и для разбивки на бэклог по командам.

## 1. Концепция продукта

Приложение для поиска людей по интересам и намерениям (например, "хочу сходить в кино", "ищу партнёра на теннис") и для организации спортивных игр и соревнований. Пользователь может найти рядом людей с похожими намерениями, создать событие (ивент), собрать участников, пообщаться в чате ивента и следить за интересующими активностями в новостной ленте.

## 2. Модули

- Events — события/ивенты
- Tags — теги (справочник + фильтрация)
- News Feed — новостная лента
- Users — регистрация, аутентификация, личный кабинет
- Chats — чат ивента
- Search — поиск по ивентам и по людям
- Geo/Map — геопоиск людей по намерениям
- User Intentions — намерения пользователей (пост-MVP)

## 3. Скоуп MVP

| Модуль | В MVP | Комментарий |

| События: создание/просмотр/редактирование/удаление/подписка | Да | Базовая сущность продукта |
| Пагинация ленты событий (по 10) | Да | Зафиксировано в заметках |
| Поиск по ивентам | Да | Postgres full-text search |
| Теги как справочник + фильтрация | Да | |
| Типы событий, пользователи могут предлагать свои | Да | Нужна модерация — см. открытые вопросы |
| Новостная лента (подписки + теги + геолокация) | Да | Комбинированный алгоритм — см. п. 6 |
| Регистрация и аутентификация (JWT + OAuth) | Да | |
| Личный кабинет | Да | |
| Групповой чат ивента (много участников) | Да | Личные 1-к-1 чаты — не в MVP |

## 4. Функциональные требования по модулям

### 4.1 Events (EventService)

- Создание ивента
- Просмотр списка ивентов с пагинацией (по 10)
- Просмотр карточки ивента
- Редактирование ивента (только автор/организатор)
- Удаление ивента (только автор/организатор)
- Подписка на ивент (вступление в участники)
- Отписка / выход из ивента
- Привязка тегов и типа события при создании
- Поиск по ивентам (текстовый поиск + фильтры: тег, тип, дата, локация)
- Типы событий — расширяемый справочник: пользователь может предложить новый тип (нужен флоу модерации, см. открытые вопросы)
- По эскизу (см. п. 11): типы событий шире, чем спорт — включают бытовые/социальные активности (кофе-встречи, бар, караоке, настолки, творчество, дискотека, группы поддержки и т.д.), не только игры
- Карточка ивента в списке/ленте: название, тип (иконка), дата, время, автор ("Создал(а): ..."), счётчик участников (например "5/6")
- Экран деталей ивента: тип, полное описание, дата/время, счётчик участников, список участников (аватар/имя), кнопка "Участвовать" ↔ "Отменить" (переключается в зависимости от статуса подписки текущего пользователя)
- Разовая платная публикация ивента: после заполнения деталей — экран "Публикация" с суммой (в эскизе пример 50 ₽) и кнопкой "Опубликовать"; после оплаты — экран-подтверждение "Опубликовано" с переходом "Смотреть карту"

### 4.2 Tags

- Теги как справочник (CRUD, вероятно админский доступ)
- Привязка тегов к ивенту при создании/редактировании
- Фильтрация ивентов по одному или нескольким тегам

### 4.3 News Feed

- Получение списка ивентов в ленту с пагинацией
- Алгоритм отбора — комбинация трёх сигналов:
  1. Подписки пользователя (организаторы/ивенты, на которые он подписан)
  2. Совпадение тегов ивента с интересами пользователя
  3. Геолокация (близость ивента к пользователю)
- Открыто: точная формула ранжирования/веса сигналов, cold start для новых пользователей без истории — см. открытые вопросы

### 4.4 Users (UserService)

- Регистрация пользователя (хранение в Postgres)
- Аутентификация: JWT (access + refresh) **и** OAuth/соцсети — оба варианта входа
- Личный кабинет: профиль, мои ивенты (созданные/подписки), настройки, интересы/теги, локация
- Локация и интересы пользователя — исходные данные для ленты и геопоиска
- По эскизу (см. п. 11), дополнительно обнаружено на макетах — статус MVP/бэклог не подтверждён, см. открытые вопросы:
  - Профиль: имя, аватар, email, город, "о себе", счётчики "Подписчиков" / "Подписок" / "Событий", кнопка редактирования по каждому полю
  - Система подписки на пользователей (фолловинг/фолловеры) — отдельная от подписки на ивент сущность
  - Уровень пользователя ("67 LVL") — геймификация
  - Рейтинг организатора/ивента (например "4.9/5⭐"), блок "Последнее событие" в карточке организатора

### 4.5 Geo/Map — геопоиск людей по намерениям

- Поиск ближайших людей на карте по совпадающим намерениям/интересам
- Требует согласия пользователя на шаринг геолокации (privacy)
- Радиус поиска, отображение на карте
- ⚠️ Зависимость: в исходных заметках `UserIntention` (создание/изменение/удаление намерения) помечено как "не в MVP", но геопоиск по намерениям выбран **в MVP**. Нужно решить, на чём именно основан геопоиск в MVP — на теговых интересах профиля (без отдельной сущности `Intention`) или на упрощённой версии намерений без полного CRUD. Зафиксировано как открытый вопрос ниже.

### 4.6 Chats

- Создание группового чата ивента (создаётся вместе с ивентом или при первом сообщении)
- Множество участников чата = участники ивента
- Отправка сообщения в чат
- Редактирование сообщения (автором)
- Удаление сообщения (автором/модератором)
- Экран "Чаты" — список всех чатов пользователя (по одному на каждый ивент), превью последнего сообщения в формате "Вы: <текст> · <время>"
- Личные 1-к-1 чаты — исключены из MVP

### 4.7 Search

- Поиск по ивентам: текст + фильтры (теги, тип, дата, локация)
- Поиск людей — через геопоиск на карте (п. 4.5)

## 5. Пост-MVP / бэклог

- `UserIntention`: создание, изменение, удаление намерения (полноценная сущность и CRUD)
- Личные 1-к-1 чаты вне контекста ивента
- Возможное уточнение геопоиска после появления полноценных `UserIntention`
- Нативные мобильные приложения (если PWA окажется недостаточно, например из-за ограничений push-уведомлений на iOS)

## 6. Микросервисы

- **EventService** — события, теги, лента, поиск по ивентам, чат ивента
- **UserService** — пользователи, аутентификация, профиль/ЛК
- **NotificationService** — уведомления, консьюмер Kafka, доставка (push/email/in-app — канал уточняется)

Организационное деление по командам (из заметок), не всегда совпадает с границами сервисов — стоит сверить на системном дизайне:

- **Группа 1**: лента событий, чат ивента, анализ отображения на карте
- **Группа 2**: события, поиск по ивентам, пользователи

## 7. Технологии

- **Frontend**: Next.js, реализуется как PWA — устанавливается на главный экран телефона (Android/iOS) и десктопа как единое веб-приложение, без отдельных нативных клиентов
- **Backend**: микросервисы на Java Spring (Maven) и Python — конкретное распределение по сервисам ещё не зафиксировано, будет описано отдельно
- **Базы данных**: 3 БД суммарно; минимум Postgres с полнотекстовым поиском для EventService — распределение по сервисам уточняется
- **Мессаджинг**: Kafka — уведомления (события → NotificationService)
- **Гео**: нужен геопоиск/гео-индекс для карты (кандидат — PostGIS поверх Postgres) — не зафиксировано

## 9. Не-цели MVP

- Отдельные нативные приложения Android/iOS — заменены PWA-версией одного Next.js-приложения
- Полноценные пользовательские намерения (`UserIntention`) как отдельная управляемая сущность
- Личные 1-к-1 чаты вне ивентов
- Монетизация/платежи — не упоминались, при необходимости добавить отдельным разделом

## 10. Зафиксированные допущения

- Пагинация списков ивентов — 10 элементов на страницу
- Аутентификация поддерживает и JWT (access + refresh), и OAuth/соцсети одновременно
- Чат ивента — групповой, участники = подписчики ивента
- Новостная лента — гибрид из подписок, тегов и геолокации, а не один сигнал
- Типы событий — открытый для пользователей справочник (с модерацией, детали уточняются)

## 11. Экраны и UI-флоу (по эскизу пользователя)

Раздел собран по ручному скетчу экранов (excalidraw). Общий дизайн-принцип автора: максимальная простота, "в духе iOS" (HIG-подобная простота элементов, минимум визуального шума). Точные связи между экранами (стрелки) в скетче не парсились попиксельно — ниже реконструкция по содержимому экранов и уже согласованной логике из разделов 1–10. Также в исходном файле обнаружен несвязанный черновик другого проекта ("Gustable", приложение для рецептов/питания) — он не имеет отношения к EventService и в спек не включён.

**Нижняя навигация** ("Кнопки навигации"): Главная/Карта, Чаты, Создать (+, в эскизе — плейсхолдер-иконка 🇨🇭 на красном квадрате), Поиск/компас, Профиль.

**Карта и поиск**
- Карта ивентов — точки на карте, фильтр "Виды активности" (иконки типов: 🏀⚽🏐🥊⛸️🎾🚴⚾🎳 — спорт, 🎤🎮🎨🧶🎲 — хобби/творчество, ☕🍻🪩🎭🫂 — социальные встречи)
- Экран "Поиск" — список/результаты ивентов, карточка: иконка типа, дата, время, автор, счётчик участников

**Создание ивента** (уточняет флоу из п. 4.1 и предыдущего наброска в чате)
1. Тап на "+" в нижней навигации
2. "Выберите тип" — сетка иконок активности
3. "Детали" — название, описание, дата/время, место
4. "Публикация" — сумма разовой оплаты (в эскизе 50 ₽) и кнопка "Опубликовать"
5. "Опубликовано" — подтверждение, кнопка "Смотреть карту" (переход на карту с фокусом на новый ивент)

**Ивент — экран деталей**
- Тип, полное описание, дата/время, счётчик участников ("3/4")
- Карточка организатора: имя, уровень ("LVL"), рейтинг ("4.9/5⭐"), "Последнее событие"
- Список участников (аватар + имя)
- Кнопка "Участвовать" ↔ "Отменить" в зависимости от статуса

**Чаты**
- Экран "Чаты" — список чатов по ивентам, превью последнего сообщения ("Вы: ... · 1 нед.")
- Чат ивента — открывается по тапу из списка

**Меню пользователя** (открывается с карты/через профиль)
- Карточка: имя, email, переключатель темы (☀️/🌙)
- Пункты: Настройки, Мои события, Интересы, Выйти

**Профиль**
- Имя, аватар, email, город, "о себе" (редактируется по каждому полю — иконки ✏️)
- Счётчики: Подписчиков / Подписок / Событий
- Кнопка "Редактировать"

**Настройки**
- Уведомления (тумблер), Геолокация (тумблер), Язык (RU)

**Мои события / Интересы** — по содержанию соответствуют экранам, уже согласованным в чате (раздел про профиль/меню выше по переписке): табы "Организую/Участвую" для событий, чипы тегов для интересов.
